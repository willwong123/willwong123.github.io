update:2014-11-7

在css文档中清楚的写道：

percentage Relative to the font size of the element itself. The computed value is this percentage multiplied by the element’s computed font size.

numberThe used value is this unitless <number> multiplied by the element’s font size. The computed value is the same as the specified <number>. In most cases this is the preferred way to set line-height with no unexpected results in case of inheritance.

可继承属性和不可继承属性

所谓继承属性，也就是父元素设定了某个属性，他的子元素会不会也因此继承到了这个属性；相反不可继承属性就是子元素不会继承父元素设定的一些属性。

继承属性可能听过也可能没有听过，但是肯定用过。只是在用的时候，并没有去认识到这是一个继承属性。比如说在页面reset.css部分，设定body的属性color：grey，那么网页中所有新建的元素字色都是grey，也就是所有的元素都继承了body元素设定的color属性。

那到底有哪些属性是属于可继承属性？有朋友整理出来如下：

不可继承的：display、margin、border、padding、background、height、min-height、max-height、width、min-width、max-width、overflow、position、left、right、top、bottom、z-index、float、clear、table-layout、vertical-align、page-break-after、page-bread-before和unicode-bidi。

所有元素可继承：visibility和cursor。

内联元素可继承：letter-spacing、word-spacing、white-space、line-height、color、font、font-family、font-size、font-style、font-variant、font-weight、text-decoration、text-transform、direction。

终端块状元素可继承：text-indent和text-align。

列表元素可继承：list-style、list-style-type、list-style-position、list-style-image。

表格元素可继承：border-collapse。

可以看到line-height也是一个可继承属性，其实这个我们应该早就知道了。同样在网页的reset部分可能也有给body设定font属性，font:<font-size>/<line-height> <font-family>。这里设定好了line-height之后，页面所有元素都会继承这个行高！

答案：line-height:1.5和line-height:150%的区别

通过上面做的结果，原来line-height:1.5和line-height:150%的确存在很大的区别！而他们的区别就在于给line-height的取值是否是一个有单位的值。

  • 当定义了一个有单位的值，如果1em或者20px或者100%的时候，浏览器会将这个元素的行高值计算出来之后传给他的全部后代元素，他的后代元素并不会拿这个有单位的值去重新计算自己的行高！

  • 而当定义了一个没有单位的值，这时候他传给后台元素的就是这个纯数字了，也就是换算系数。他的后代元素会拿这个系数乘以自己的字号值得到实际的行高值。
