替换元素与非替换元素

  2014-11-19

早上过来，照例打开知乎看看有什么新鲜东西。无意中见到一个很简单的提问，我也就点进去看了。问题是这样的：

html页面中的img元素是块级元素还是行内元素？如果是行内元素为什么可以设置宽高，如果是块级元素，为什么多张图片是并排排的？

这一下子让我慌了！往下面看，发现有人指出来了，原来img元素是行内可替换元素！

于是开始了各种查资料，看博客；了解到原来在html元素中，除了行内元素和块级元素分类之外，还是有可替换元素和不可替换元素两个分类方式。

  • 可替换元素：类似于img,canvan,textarea,input这些元素，他们的元素本身是在页面中不占有空间的，就是一个空元素。只是当这些元素引用了特定的属性内容时，浏览器会根据属性给元素一定的尺寸，这些元素才会在页面中占有一定的空间。

如果在页面中出现只写一个<input />这种标签，在浏览器中我们是看不到的；但是如果给input添加一个属性type="text"变成<input type="text"/>之后，这个元素在浏览器中便会显示出来了！

  • 不可替换元素：不可替换的元素就是我们常见的一些页面中会占有默认空间的元素，像p无论你内容有什么，都会在页面中直接显示出来元素的内容。

html中绝大多数的元素都是不可替换元素！

W3C的说明

这段内容是W3C对替换元素在网页中特别的展示效果的说明。

If ‘height’ and ‘width’ both have computed values of ‘auto’ and the element also has an intrinsic width, then that intrinsic width is the used value of ‘width’.

If ‘height’ and ‘width’ both have computed values of ‘auto’ and the element has no intrinsic width, but does have an intrinsic height and intrinsic ratio; or if ‘width’ has a computed value of ‘auto’, ‘height’ has some other computed value, and the element does have an intrinsic ratio; then the used value of ‘width’ is:(used height) * (intrinsic ratio)

If ‘height’ and ‘width’ both have computed values of ‘auto’ and the element has an intrinsic ratio but no intrinsic height or width, then the used value of ‘width’ is undefined in CSS 2.1. However, it is suggested that, if the containing block’s width does not itself depend on the replaced element’s width, then the used value of ‘width’ is calculated from the constraint equation used for block-level, non-replaced elements in normal flow.

Otherwise, if ‘width’ has a computed value of ‘auto’, and the element has an intrinsic width, then that intrinsic width is the used value of ‘width’.

Otherwise, if ‘width’ has a computed value of ‘auto’, but none of the conditions above are met, then the used value of ‘width’ becomes 300px. If 300px is too wide to fit the device, UAs should use the width of the largest rectangle that has a 2:1 ratio and fits the device instead.

  • 当这个替换元素设定宽度和高度都是auto，替换元素就会使用该元素自身自带的高宽度；
  • 如果宽度或者高度都是设定为autp，元素中没有自带的宽度，但是有自带高度和比例，那么元素会将自带高度乘以自带的比例来生成该元素的宽度；相反的，如果不知道高度的时候，用宽度乘以比例。
  • 如果只知道元素的比例，那么该元素的尺寸会强制等于普通文档流中的块级非替换元素的尺寸；
  • 如果计算出元素的宽度是auto，元素有一个自身宽度，那么宽度就是元素自身的宽度；
  • 否则，如果上面的情况都没有，并且元素的宽度设定的是auto，那么元素的宽度就设定为300px，如果宽度300px超出屏幕尺寸的话，那么UA将会采用2:1比例下最大的矩形适配屏幕。
