TO BE STRONGER

前端工具yo+bower+grunt自动化

  2015-08-31

以前开发一套页面的时候，总要花点时间去准备思考文档目录结构怎么构建。虽然可能有一套大家默认的结构，样式文件就放在css目录，教程就放在js目录。但是有会出现另一个问题，这些静态资源在上线的时候需要压缩，输出的方法再单独重新压缩？

现在的整合

这些应该就是以前开发中的步骤。每个步骤都是单独的，互相独立的，每次这些事情也都会占用我们这些本来不多的切图时间。更有说不定的，不太会用，导致这些时间占了大头！

所以，当nodejs火起来，这些问题也可以得到解决了。让我们可以很多时间去做我们该做的事，让这些复杂，没有技术含量的活交给计算机程序。

前端开发自动化

此时，yeoman, bower, grunt等一套工具就出现了。必然的，都是依赖于nodejs。

安装

首先要在电脑上安装nodejs，这里不多说，自行搜索。

接着用npm安装上面说的这些工具

npm install -g yo bower grunt-cli

这里的安装都是全局的，不过光安装这些还不够。yo只是一套脚手架，具体你找搭出什么形状的还需要一个生成器generator，在yo网站上可以看到很多目前已经存在的generators，我们可以根据实际需要选择安装。
例如: 要安装一套bootstrap的脚手架

npm install -g generator-bootstrap

这样就安装好了bootstrap的样式了，接着就在自己的文件目录下面执行yo bootstrap就可以在当前目录下生成一套以bootstrap构建的文件了，接着会自动安装需要的文件。完成之后，里面会出现bower_modules和node_modules两个文件夹，这里面都是需要的文件。

如果我们不需要再添加什么插件了，就可以跳过bower了，直接输入grunt serve，就可以在看到在浏览器中打开了我们部署的文件。

可以看到，grunt执行了压缩，打包，校验等等工作，并实时刷新页面。

接下来就可以自由开发了！
